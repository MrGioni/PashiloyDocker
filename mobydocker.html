Moby Docker

Что такое Moby и как он связан с Docker? – Облачные вычисления

Проект Docker Moby разбирает основные компоненты Docker Engine в модульный набор инструментов, который могут повторно использовать другие системы на основе контейнеров. Moby был создан на основе тогда еще монолитной кодовой базы Docker в 2017 году. Он превратился в обширную библиотеку серверных компонентов контейнера, которые можно комбинировать для создания полных контейнерных решений, таких как сам Docker.

Как конечный пользователь Docker вы не взаимодействуете напрямую с проектом Moby. Он предназначен для людей, создающих платформы контейнеризации, а не для разработчиков, создающих и запускающих образы контейнеров. Однако вы можете столкнуться с Moby в документации Docker или при подаче отчетов об ошибках и запросов функций.

Докер перед Моби

Изначально Docker разрабатывался в кодовой базе, которая содержала все необходимое для проекта. Это варьировалось от среды выполнения контейнеров и сборщиков образов до поставщиков хранилищ, управления сетью и интерфейса командной строки.

Когда в середине прошлого десятилетия популярность Docker резко возросла, было замечено, что этот подход «все в одном» сдерживает более широкую экосистему. Дополнительные инструменты не могли полагаться на конкретные вещи Docker, потому что ничто не было компонентом. Внешним поставщикам пришлось использовать всю разросшуюся платформу Docker.

Базовые функциональные блоки, такие как containerd, были быстро разделены на автономные модули. Теперь сообщество может создавать новые системы контейнеров, не изобретая заново среду выполнения, которая взаимодействует с ядром для запуска экземпляров контейнеров. Затем последовали другие компоненты, такие как runc и HyperKit, которые были отделены от проекта Docker, а затем повторно интегрированы в выпуски Docker Engine в виде модульных зависимостей.

Moby поднял этот подход на новый уровень, извлекая из Docker еще больше компонентов. Старый docker/docker Репозиторий GitHub стал moby/moby; он содержит части кода Docker Engine с открытым исходным кодом в доступном для сообщества месте, независимо от репозиториев продуктов Docker CE и EE с закрытым исходным кодом.

Сборочная модель

С Moby каждый может смешивать и сочетать части технологии Docker без необходимости тянуть всю партию. Он предоставляет реализации всех ключевых подсистем, составляющих платформу контейнеризации. Вы можете создать свою собственную версию Docker, а затем заменить ее, чтобы использовать альтернативные проекты в определенных функциональных областях.

Подход Moby основан на использовании общих платформ в автомобильной промышленности. Marques строит целые семейства моделей из одного ряда шасси и двигателей. Добавление новой опции в модельный ряд достигается за счет использования общей платформы и ее настройки в соответствии с требованиями дизайна нового автомобиля. Платформа не обязательно будет использоваться как есть: модель, которая будет предлагаться в качестве выбора производительности, может иметь варианты двигателя и модификации шасси, которые недоступны на других моделях, использующих платформу.

Moby предлагает нечто подобное разработчикам контейнерных систем. Вы можете начать с предопределенных сборок компонентов, которые обеспечивают проверенную кросс-совместимость. Это основная общая автомобильная платформа. По мере развития вашей системы вы заменяете отдельные модули новыми реализациями, чтобы создать более мощный инструмент. Примером может быть замена генератора образов BuildKit на более мощную альтернативу. Автопроизводитель должен заменить стандартный небольшой двигатель своей платформы на альтернативу большей мощности.

Таким образом, опыт разработчика Moby заключается в сборке и расширении. Каждый проект на основе Moby будет использовать различный набор частей для достижения уникальной цели. До Moby у разработчиков не было доступной отправной точки для инноваций в определенных областях экосистемы контейнеров. Теперь разработки могут быть более целенаправленными, когда Моби заполняет пробелы.

От Моби к Докеру

Хотя Moby включает в себя все основы контейнерной системы, упор на разработчиков системы означает, что установка эталонной сборки не даст вам наилучших результатов. Именно здесь дополнительные проекты, такие как Docker, добавляют ценность. Они опираются на Moby, чтобы создать платформу, которая будет полезна для конечных пользователей, работающих с контейнерами и образами.

Например, установите Docker и запустите docker pull ubuntu:latest возможно без какой-либо ручной настройки. Это связано с тем, что Docker предварительно настроен на извлечение неквалифицированных ссылок на образы из Docker Hub. Это не будет работать с эталонной сборкой Moby; хотя он может извлекать изображения, у него нет регистра по умолчанию, поэтому он не сможет разрешить тег.

Нижестоящие платформы добавляют мнения к объективному набору функций, предлагаемых Moby. Хотя разработчикам контейнерных систем не нужен предварительно настроенный URL-адрес реестра, это полезная возможность для создателей образов. До появления Moby у разработчиков платформы не было возможности фильтровать решения Docker, основанные на продуктах.
